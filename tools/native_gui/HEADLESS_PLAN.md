# Headless / Agent Automation Plan for tools/native_gui

## Goal
Enable deterministic, scriptable runs of the ImGui native harness so AI agents (and CI) can control inputs, capture pixel-perfect screenshots, and read structured engine outputs to automate debugging and visual validation.

## Scope
- Add a `--headless` mode to `gui_main.cpp` that can run without interactive UI and produce artifacts (PNG + JSON).
- Provide deterministic stepping and optional seed control so frames are reproducible.
- Expose a minimal HTTP/JSON control API to let external agents set inputs and request screenshots/outputs.
- Provide runner scripts and image-diff tests for CI.

## CLI (suggested)
- `--headless`           : run without interactive window
- `--preset <file>`      : load preset JSON
- `--frames <N>`         : number of frames to run
- `--screenshot-every K` : save a screenshot every K frames
- `--out-dir <path>`     : directory for PNG/JSON outputs
- `--seed <N>`           : RNG seed for determinism
- `--no-window`          : create D3D device off-screen (optional)
- `--http-port <P>`      : enable HTTP control on port P

## High-level Implementation Steps
1. Parse CLI early in `WinMain` and set a `headless` mode flag.
2. Add deterministic timing: use fixed-step loop (FRAME_STEP_US) and seed any RNGs from `--seed`.
3. Implement screenshot helper (copy swapchain/backbuffer -> staging texture, map, write PNG). Recommend `stb_image_write.h` for minimal dependency.
4. Serialize `g_display_sol` + relevant `g_state` fields to JSON per requested frame (using existing `nlohmann::json`).
5. Support off-screen rendering path (hidden window + swapchain copy) first; optionally add true headless D3D11 path later.
6. Add a simple HTTP/JSON control server (single-header `cpp-httplib` or custom TCP JSON) exposing endpoints: `/load_preset`, `/set_input`, `/step?n=`, `/screenshot`, `/state`.
7. Add `tools/native_gui/runner.ps1` and `runner.sh` to exercise cases and run pixel diff against `tools/native_gui/testdata/golden/`.
8. Add image-diff tool (ImageMagick `compare` or a small C++ pixel-compare with tolerance) and CI job configuration in `platformio.ini` or CI pipeline.
